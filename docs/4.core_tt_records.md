# Records in core_tt

Record types are structural `struct` values with field declarations. Record instances are separate runtime values constructed from those struct type values.

## Construction

```fun
let Point = struct
  x: I64;
  y: I64;
end in
Point {x = 3; y = 4}
```

Construction checks that every required field is present, no unknown fields are supplied, and every field value checks against the declared field type. Field order at construction sites is not semantically significant.

Parameterized record-like structs are represented as functions returning struct values; implicit insertion handles common construction and field-access cases.

## Field access

Record instances and module values both use dot syntax:

```fun
p.x
M.value
```

For records, dot access reads an instance field. For modules, dot access reads a public member. Passing modules through functions requires an explicit module signature when the function accesses module members.

## Patterns

Record patterns name the record type and destructure fields by name:

```fun
match p with
| Point {x; y} -> x + y
end
```

Supported forms include:

```fun
Point {x; y}
Point {x = n; y}
Point {x; _}
Point {flag = true; value}
M.Point {x; y}
```

A complete record pattern must list every field. A partial pattern uses trailing `_` to allow omitted fields.

## Runtime and matching model

Record construction evaluates to a record instance value carrying its type value and field values. Pattern compilation introduces named field occurrences so nested field subpatterns can reuse the same decision-tree machinery as tuple and constructor matching.
