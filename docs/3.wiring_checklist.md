# Core implementation checklist

This checklist tracks current `core_tt` capability rather than migration from an older implementation path.

## Implemented

- [x] Bidirectional elaboration with NbE-backed conversion
- [x] Implicit arguments and implicit insertion
- [x] Let-generalization through implicit parameters
- [x] Primitive atoms: `I64`, `Bool`, `Unit`, `Char`
- [x] Arithmetic and comparison primitives
- [x] `let rec` / `Fix` in surface syntax and core evaluation
- [x] Nominal ADTs, including parameterized and recursive ADTs
- [x] Structs as modules/namespaces
- [x] Struct field declarations as record types
- [x] Record construction and field access
- [x] Record patterns, including renamed and partial field patterns
- [x] Tuple/product values and patterns
- [x] Or-patterns
- [x] Qualified constructor/type paths in patterns
- [x] Decision-tree match compilation and exhaustiveness checking for finite domains
- [x] Type-head patterns for type-directed specialization
- [x] Polymorphic `==`/`!=` through type specialization
- [x] `method`/`self`/`Self` in struct record types
- [x] Core module-file imports with caching and circular import detection
- [x] Main executable uses the core parse/elaborate/evaluate pipeline
- [x] Main semantic/runtime regression coverage has been ported to core tests where supported by the type system

## Open or lower priority

- [ ] Broaden final user-facing parser syntax beyond the current core grammar
- [ ] Improve diagnostics for parse, elaboration, unification, and exhaustiveness failures
- [ ] Decide the long-term design for equality over nominal and record values
- [ ] Replace `Type : Type` with a universe hierarchy if the language needs consistency
- [ ] Design the next runtime/backend layer after the NbE interpreter
