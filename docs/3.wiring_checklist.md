# Wiring Checklist: Features needed before connecting to old pipeline

## Done

- [x] `let rec` / `Fix` in surface syntax and core evaluation
- [x] Decision-tree match compilation and exhaustiveness checking for nominal ADT constructor patterns
- [x] Literal patterns for `I64`, `Bool`, and `Unit`

## Blocking

- [x] Char literal patterns — single-quoted `Char` literals in expressions and match branches
- [x] Tuple patterns — destructuring `(a, b, c)` in match branches
- [x] Or-patterns — `A | B -> ...` combining alternatives
- [x] Record patterns — `Foo {x; y}` or `Foo {x; _}` destructuring ([plan](4.core_tt_records.md))

## Probably blocking

- [x] Polymorphic `==`/`!=` — specialize on implicit type params for now, with monomorphic equality primitives underneath ([plan](5.core_tt_type_specialized_equality.md))
- [x] Parameterized struct record types — modeled as functions returning structs, with implicit insertion for record construction and `Pair.field` access
- [ ] Qualified constructor/type paths in expressions and patterns — old pipeline supports `M.Red`, `A.B.X(n)`, and aliases like `N.Red`; core_tt mostly relies on unqualified names via scope/open
- [x] Struct variant syntax parity — intentionally not carried forward: structs are structural, ADTs are nominal; translate old `let Color = struct | Red | Green end` cases to core_tt `type Color = Red | Green in ...`
- [ ] Recursive `self` in struct variants — old pipeline supports `struct['a] | Cons ('a, self) | Nil end`; core_tt has recursive nominal definitions but not this surface form
- [ ] Translate main semantic/runtime regression tests to core_tt equivalents — constants/lets/recursion/matches/records mostly translate, but no shared test corpus exists yet

## Lower priority

- [ ] Record-type-directed field inference — partial-struct unification covers some field access cases, but not old-pipeline named-record inference
- [ ] Main-pipeline record declaration syntax compatibility — old pipeline accepts `type point = {x: I64; y: I64}`; core_tt intentionally models record types as `struct` values
- [ ] Top-level file/import pipeline for core_tt — old pipeline tests import resolution/circular import detection; core_tt is still expression/test-parser only
- [ ] Main parser syntax parity — old parser tests cover comments, `;;`, function-binding sugar, import syntax, and type-variable syntax that core_tt's test parser does not currently target
