# Type-case / generic programming plan

## Goal

Add useful generic programming over runtime `Type` values by extending existing primitive type-head matching with branch-local dependent refinement. The first milestone should let a match on `T : Type` refine values whose type depends on `T`, while keeping nominal type-head matching for a later phase.

## Decisions

- Type-case is a normal runtime match over `Type` values.
- Matches over open `Type` require a fallback because the type universe is infinite.
- The first implementation phase focuses on primitive type heads: `I64`, `Bool`, `Unit`, and `Char`.
- Nominal type-head matching is planned separately after primitive refinement works.
- In a primitive type-head branch, the checker may treat the matched type variable as equal to the matched primitive type.

## Phase 1: Primitive dependent refinement

Implement branch-local refinement for matches of the form:

```fun
let is_zeroish : (T : Type) -> T -> Bool = fun T x ->
  match T with
  | I64 -> x == 0
  | Bool -> x == false
  | Unit -> true
  | Char -> x == '\000'
  | _ -> false
  end
```

Expected behavior:

- In the `I64` branch, `T` refines to `I64`, so `x : T` is usable as `I64`.
- In the `Bool` branch, `T` refines to `Bool`, so `x : T` is usable as `Bool`.
- In the `Unit` branch, `T` refines to `Unit`.
- In the `Char` branch, `T` refines to `Char`.
- The fallback branch does not refine `T`.

Implementation notes:

- Keep runtime semantics as ordinary match evaluation over `Type` values.
- Reuse existing primitive type-head pattern support where possible.
- Add elaboration-time refinement only while checking the selected branch body.
- The refinement should affect dependent variables in scope whose type mentions the matched type variable.
- Do not add nominal type matching, open rows, protocols/typeclasses, or macro syntax in this phase.

Tests:

- `is_zeroish I64 0` evaluates to `true`.
- `is_zeroish I64 1` evaluates to `false`.
- `is_zeroish Bool false` evaluates to `true`.
- `is_zeroish Bool true` evaluates to `false`.
- `is_zeroish Unit ()` evaluates to `true`.
- `is_zeroish Char '\000'` evaluates to `true`.
- A fallback branch is required for an arbitrary `T : Type` scrutinee.
- A branch should reject operations that do not typecheck under its refinement.

## Phase 2: Refined return types

Support type-case expressions whose branch result is checked against a type depending on the matched type variable.

Example:

```fun
let default : (T : Type) -> T = fun T ->
  match T with
  | I64 -> 0
  | Bool -> false
  | Unit -> ()
  | Char -> '\000'
  | _ -> ?
  end
```

This phase needs a policy for the fallback branch because there is no total default value for every possible type in the current language. Options include:

- keep this example rejected until a bottom/absurd/option mechanism exists;
- require callers to provide a fallback value;
- return an explicit option-like result once suitable library support exists.

## Phase 3: Nominal type heads

Extend type-case refinement to nominal type heads after primitive heads are stable.

Open questions for this phase:

- Syntax for matching parameterized nominal heads.
- Whether branch patterns can bind nominal parameters.
- How nominal record and ADT IDs are compared at runtime and elaboration time.
- How much refinement should be exposed for constructor/result values inside the branch.

Example target shape:

```fun
type Option A = Some A | None in
let classify : Type -> I64 = fun T ->
  match T with
  | I64 -> 1
  | Option -> 2
  | _ -> 0
  end
```

## Phase 4: Generic programming library surface

Once primitive and nominal refinement are in place, add small library-style examples that justify the feature:

- type-directed equality dispatch;
- type-directed defaulting where a fallback value is supplied;
- type-directed serialization/classification once strings or richer data types exist;
- protocol/typeclass bridge experiments after the protocol design exists.

## Non-goals for now

- Macro integration.
- Protocol/typeclass implementation.
- Nominal type-head matching in the first milestone.
- Compile-time-only type-case erasure.
- Exhaustive matching over all `Type` values.
- A universal default value for arbitrary types.
